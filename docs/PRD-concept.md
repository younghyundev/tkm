# Tokenmon — Conceptual PRD

> Claude Code를 사용할수록 포켓몬이 성장하는 경험치 플러그인

## 1. Vision

**"코딩하면서 포켓몬을 키운다."**

Tokenmon은 Claude Code의 토큰 소비를 경험치로 변환하여, 개발자가 일하는 동안 자연스럽게 포켓몬을 성장시키는 게임 레이어다. 별도의 조작 없이 평소대로 코딩하면 포켓몬이 레벨업하고, 진화하고, 새로운 포켓몬을 만난다.

## 2. Core Loop

```
코딩 (토큰 소비)
    ↓
XP 획득 (파티 전원)
    ↓
레벨업 → 진화 체크
    ↓
야생 포켓몬 조우 (15% 확률)
    ↓
전투 → 승리 시 포획 + EV 획득
    ↓
업적 달성 → 새 포켓몬 해금
    ↓
지역 이동 (포켓도감 진행도)
    ↓
반복
```

모든 게임 이벤트는 **Claude Code 세션 종료 시점**에 일괄 처리된다. 사용자는 코딩에 집중하고, 세션이 끝나면 결과를 확인한다.

## 3. Target User

- Claude Code를 일상적으로 사용하는 개발자
- 포켓몬에 대한 향수 또는 관심이 있는 사용자
- 반복적인 개발 작업에 소소한 재미를 더하고 싶은 사용자

## 4. Design Principles

### 4.1 Zero Friction
- 설치 후 별도 조작 불필요 — 훅이 자동으로 게임 이벤트 처리
- StatusLine에 실시간 상태 표시 — 별도 명령 없이 현재 상태 파악
- 진화/전투/포획 모두 자동 — 사용자가 개입할 필요 없음

### 4.2 Authentic Pokemon Feel
- Gen 4 (신오 지방) 107종 포켓몬, 한글 이름
- 원작 경험치 공식 6종 (medium_fast, medium_slow, slow, fast, erratic, fluctuating)
- 18종 타입 상성 시스템
- 레벨/우정/지역/아이템 기반 진화 조건
- 울음소리 + ANSI 스프라이트 (Braille/Terminal art)

### 4.3 Balanced Progression
- 토큰 소비 → XP 변환 비율 조정 가능 (기본 10,000:1)
- 전투 승률은 타입/레벨/스탯/EV 4요소로 결정, 항상 3%~95% 범위
- 파티 멀티플라이어 최대 1.5x (6인 파티 기하급수 가중치)
- EV는 최대 1.252x 보정 — 레벨/타입보다 작은 영향력

### 4.4 Developer-Friendly
- CLI 기반 인터페이스 (`/tkm status`, `/tkm party`, `/tkm pokedex`)
- 치트 명령어로 디버깅 가능 (`/tkm cheat xp 10000`)
- Doctor 스킬로 9단계 설치 검증
- 모든 데이터는 JSON으로 투명하게 저장

## 5. Game Systems

### 5.1 스타터 선택
최초 설치 시 3종 중 택1:
- **모부기** (풀/땅) — 방어 특화, medium_slow
- **불꽃숭이** (불꽃/격투) — 공격 특화, slow  
- **팽도리** (물/강철) — 균형, medium_fast

### 5.2 야생 조우 & 전투
- 세션 종료 시 15% 확률로 야생 포켓몬 조우
- 파티 중 최적 포켓몬이 자동 선택되어 전투
- 승률 = 타입 상성 × 레벨 차이 × 스탯 비율 × EV 보정
- 승리 시: 포획 + 파티 전원 EV +1 + XP 보상
- 패배 시: 자동 재도전 (retry token 보유 + 승률 > 60%)

### 5.3 진화
포켓몬이 조건을 충족하면 자동 진화:
- 레벨 도달 (예: 모부기 Lv.18 → 수풀부기)
- 우정도 220 이상 (예: 리오르 → 루카리오)
- 특정 업적 달성 = 교환 진화 대체
- 아이템 보유, 특정 지역, 특정 기술 습득

### 5.4 지역 시스템
포켓도감 진행도에 따라 새 지역 해금:
- 9개 신오 지역, 각각 레벨 범위 + 고유 포켓몬 풀
- 해금 조건: 포켓도감 발견/포획 수 (예: 5마리 포획 → 풀밭지방)
- 지역별 레벨 범위가 조우 포켓몬 레벨 결정

### 5.5 업적 시스템
특정 마일스톤 달성 시 보상:
- 포켓몬 해금 (예: 첫 세션 → 팽도리, 첫 에러 → 찌르꼬)
- 아이템 지급
- 파티 크기 확장 (50회 권한 허용 → max 6)
- 25+ 업적, 3단계 희귀도 (★☆☆, ★★☆, ★★★)

### 5.6 서브에이전트 파견
- 서브에이전트 생성 시 파티 포켓몬 1마리 자동 파견
- 파견 포켓몬은 1.5x XP 멀티플라이어 적용
- `default_dispatch` 설정으로 우선 파견 포켓몬 지정 가능

## 6. User Journey

### Day 1: 설치 & 스타터
1. `npm install` → 자동 설정
2. `/tkm setup` → StatusLine 등록 + 스타터 선택
3. StatusLine에 선택한 포켓몬 스프라이트 표시

### Week 1: 성장
- 일상적 코딩으로 XP 축적 → 레벨업
- 첫 업적 달성 → 새 포켓몬 해금
- 첫 야생 조우 → 전투 경험
- 첫 진화 이벤트

### Month 1: 탐험
- 여러 지역 해금 → 다양한 포켓몬 조우
- 파티 구성 최적화 (타입 커버리지)
- EV 축적으로 전투력 강화
- 포켓도감 완성 도전

## 7. Monetization & Distribution

- **무료 오픈소스** — npm 패키지로 배포
- Claude Code 플러그인 마켓플레이스 등록 예정
- 수익화 모델 없음 — 순수 재미 목적

## 8. Success Metrics

| 지표 | 측정 방법 |
|------|----------|
| 설치 수 | npm download count |
| 리텐션 | session_count 분포 |
| 진행도 | 평균 포켓도감 완성률 |
| 밸런스 | battle_wins / battle_count 비율 (목표: 40-60%) |

## 9. Roadmap (Backlog)

- **체육관 전투** — 지역별 연속 전투 챌린지
- **기술/특성 시스템** — 전투 다양성
- **분기 진화** — 조건별 다른 진화 경로
- **상태이상** — 독/마비/잠듦 등
- **소지 아이템** — 전투/진화 영향
- **리트라이 토큰 경제** — 획득/소비 밸런스 조정
